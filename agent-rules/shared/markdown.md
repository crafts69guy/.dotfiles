# Markdown Rules

Use these rules for durable Markdown that may be read by humans and agents.

- Use clear headings and short paragraphs.
- Prefer concise bullets for checklists and decisions.
- Use fenced code blocks with language labels.
- Keep tables short; use lists when tables become dense.
- Do not duplicate metadata in body text when the target system has native metadata fields.
- Avoid fragile Markdown inside renderer-sensitive formats such as Mermaid labels.
- Keep examples separate from rules when examples are long.
