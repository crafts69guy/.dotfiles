# Smart Commit

Help me create a git commit with a well-crafted message.

## Instructions

1. First, run `git status` to show me all changed files (staged and unstaged)
2. Run `git diff --staged` to see what's already staged, and `git diff` to see unstaged changes
3. Present me with a list of changed files and ask which ones I want to include in this commit (use AskUserQuestion with multiSelect)
4. Stage the selected files using `git add`
5. Analyze the staged changes and generate a commit message following these rules:
   - Use conventional commit format: `type(scope): description`
   - Types: feat, fix, refactor, chore, docs, style, test, perf
   - Keep the description under 50 characters
   - Focus on WHAT changed and WHY, not HOW
   - Be concise and specific
6. Show me the proposed commit message and ask for confirmation
7. If confirmed, execute the commit

## Commit Message Style

- Prefer active voice: "add feature" not "added feature"
- No period at the end
- Lowercase after the colon
- Do NOT include "Co-Authored-By", "Generated by", or any AI attribution
- Keep the commit message clean and simple (subject line only)
- Examples:
  - `feat(auth): add password reset flow`
  - `fix(api): handle null response in user endpoint`
  - `refactor(utils): simplify date formatting logic`
  - `chore(deps): update typescript to 5.3`
